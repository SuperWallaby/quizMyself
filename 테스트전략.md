다음과같은 테스트가 필요함

# 만들기

1. 타입별 퀴즈 만들기
2. 추가 옵션 테스트
3. 실패시 테스트

# 리스트

1. 퀴즈가 리스트에 있는지 확인
2. 퀴즈를 삭제할수 있음
3. 퀴즈를 페이지네이션 할수 있음

# 풀기 테스트

1. 퀴즈가 제대로 화면에 나타나는지 확인
2. 정답이 일치하는지 확인
3. 테스트 결과가 정확한지 확인

# 설정테스트

1. 다국어가 제대로 반영되는지 확인.
2. 퀴즈 순서가 제대로 나오는지 확인
3. 퀴즈 제한 갯수가 일치하는지 확인

# 테스트전략

fixture에 사용할 퀴즈들을 저장해둠
각 테스트마다 IndexDB는 초기화됨
해당 퀴즈마다 fixture 에 있는 퀴즈들을 꺼내와 IndexedDB에 저장해줌.
IndexDb 에 저장된 내용을 바탕으로 다음 테스트 작업들을 수행함
